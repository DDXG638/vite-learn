# 第11章：Vite 与 Webpack 对比

## 本章概述

本章将对比 Vite 和 Webpack 两大构建工具，从构建速度、热更新机制、配置复杂度等多个维度进行分析，帮助你选择适合自己项目的构建工具。

## 依赖包说明

| 依赖包 | 版本 | 说明 |
|-------|------|------|
| vite | ^7.0.5 | 核心构建工具 |
| @vitejs/plugin-vue | ^6.0.5 | Vue 插件 |
| vue | ^3.5.13 | Vue3 框架 |

## 运行项目

```bash
# 安装依赖
pnpm install

# 开发模式
pnpm dev

# 生产构建
pnpm build

# 预览构建结果
pnpm preview
```

## 知识点

### 1. 构建速度对比

**Vite**:
- 使用 esbuild 进行依赖预构建
- 开发时使用原生 ESM，不需要打包
- 按需加载，只处理当前页面需要的模块

**Webpack**:
- 使用 babel-loader 或 ts-loader 进行转译
- 全部代码需要打包后才能运行
- 大型项目首次构建较慢

### 2. 热更新机制对比

**Vite**:
- 基于 ESM 的热更新
- HMR 只更新变化的模块
- 保持应用状态不丢失
- 速度极快

**Webpack**:
- 需要重新构建
- 热更新速度与项目规模成正比
- 大型项目热更新可能较慢

### 3. 配置复杂度对比

**Vite**:
- 配置简洁，默认配置已满足大部分需求
- 开箱即用
- 官方插件质量高

**Webpack**:
- 配置复杂，需要大量手动配置
- 需要配置各种 loader
- 插件配置繁琐

### 4. 构建原理对比

**Vite**:
- 开发模式：原生 ESM + esbuild 预构建
- 生产模式：Rollup 打包
- 开发与生产使用不同构建策略

**Webpack**:
- 全程使用 Bundless 打包
- 所有代码打包成 bundle
- 开发与生产使用相同构建策略

### 5. 生态对比

**Vite**:
- 较新的构建工具
- 插件生态正在发展中
- 与 Rollup 插件兼容

**Webpack**:
- 成熟稳定的构建工具
- 插件生态丰富
- 大量成熟解决方案

### 6. 适用场景

**Vite 适合**:
- 现代化前端项目
- 中大型项目
- 需要快速启动的项目
- Vue 3 / React 18 项目

**Webpack 适合**:
- 老项目迁移
- 复杂构建需求
- 需要高度自定义的项目
- IE11 支持

### 7. 迁移注意事项

从 Webpack 迁移到 Vite 时注意：
- 检查是否使用了 Webpack 特有插件
- 确认 CSS 处理方式是否兼容
- 检查环境变量使用方式
- 注意动态 import 语法差异
